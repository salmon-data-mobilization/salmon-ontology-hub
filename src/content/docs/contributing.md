---

## Ways to contribute

- Suggest new terms or definitions
- Improve documentation and examples
- Report issues or gaps
- Participate in reviews

## Workflow

1. Open an issue describing the proposed change.
2. If approved, submit a pull request with updates.
3. Use clear, concise language and link to supporting evidence where possible.

## Writing style

- Keep terminology consistent with the ontology.
- Prefer plain language, with citations for domain-specific claims.
- Use short sections and headings to make scanning easy.

## Need help?

Reach out via GitHub issues: <https://github.com/salmon-data-mobilization/salmon-ontology-hub/issues>
